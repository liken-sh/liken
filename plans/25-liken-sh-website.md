# A website on liken.sh

Milestone 25 — Not started

The domain already exists: liken.sh is the project's name in the one
namespace that matters to the code (every CRD lives under liken.sh/
v1alpha1), and it should answer in the namespace that matters to
people. This milestone is the site itself: what liken is, why it
exists, and where to start reading.

The repo's whole conceit is that it is written to be read, so the
site's first job is to say so and point inward, not to duplicate the
writing that already lives next to the code. A small, fast, static
site fits the project's temperament: no framework the reader has to
believe in, nothing to operate, the same plainness the OS itself
aims for.

Open questions, deliberately unanswered here: how the site is built
and hosted (a static generator in the repo, or hand-written pages —
either way it should live in this repo, organized as its own domain);
what the front page owes a first-time reader that the plans' README
doesn't already say; and how much of the milestone story belongs on
the web versus in the repo that tells it.
